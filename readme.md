### 一、后端代码生成需求（核心优化+实时交互）
#### 开发背景
基于已有的学生hub系统（Spring Boot 3.5.5 + MyBatis-Plus 3.5.7 + MySQL 8.0 + Redis + JWT + 阿里云OSS + 邮件发送 + WebSocket），已完成用户表、帖子表、公告表、评论表、点赞表及对应CRUD，需生成**剩余未完成功能的完整后端代码**，核心优化「数据权限隔离」「WebSocket实时提醒」「学生私信交互」逻辑，要求如下：

#### 核心要求
1. 代码规范：包结构统一为`com.it.zwx.studenthub_system`，遵循Controller→Service→Mapper→Entity（DTO/VO分层）分层，使用Lombok简化代码，MyBatis-Plus实现数据库操作；
2. 适配配置：严格匹配提供的application.yml（MySQL/Redis/邮件/JWT/MyBatis-Plus/WebSocket配置），无需额外配置；
3. 功能完整：覆盖学生/辅导员/管理员未完成的所有核心需求，包含表结构SQL、实体类、DTO、Mapper、Service（Impl）、Controller、异常处理、工具类、WebSocket核心逻辑等；
4. 兼容性：支持Java 17，LocalDateTime序列化正常，分页使用PageHelper，OSS上传适配阿里云SDK，JWT鉴权集成到接口，WebSocket实现全场景实时提醒；
5. 易用性：代码包含清晰注释，接口返回统一Result格式（已存在），关键功能（请假审批、打卡上传、WebSocket推送）包含完整业务逻辑校验；
6. 数据隔离：学生仅能查看本人的课程/考试/打卡/请假数据，辅导员仅能查看所负责班级学生数据，管理员可查看全量数据。

#### 数据库表结构优化设计（必须严格执行）
1. 学生表（扩展原有用户表）：
   - 新增字段：`class_no` varchar(50)（班级编号，如：计科2023-1班）、`role` tinyint（角色：1-学生 2-辅导员 3-管理员）、`counselor_id` bigint（所属辅导员ID）；
2. 课程表（course）：
   - 核心字段：ID、课程名称、课程编号、授课老师、上课时间/教室、学期、`class_no`（关联班级编号）、状态、创建/修改时间；
   - 索引：按班级编号建索引，支持学生按班级查询本人课程；
3. 考试表（exam）：
   - 核心字段：ID、课程ID、考试名称、考试时间、地点、提醒时间、`class_no`（关联班级编号）、状态、创建/修改时间；
   - 索引：按班级编号、考试时间建索引，支持学生查询本人考试+系统提醒；
4. 消息表（优化，支持私信+系统提醒）：
   - 核心字段：ID、发送人ID、接收人ID、消息类型（1-系统提醒 2-私信 3-互动提醒）、消息内容、是否已读（0-未读 1-已读）、发送时间、关联业务ID（如请假ID/帖子ID）；
5. 打卡表/请假表：保留原有字段，新增`class_no`（班级编号），支持辅导员按班级筛选。

#### 需生成的功能模块（按优先级）
##### （一）基础通用组件（先生成）
1. 通用返回结果类（Result）：复用已有，无需重复生成；
2. 自定义业务异常类（BusinessException）：继承RuntimeException，用于业务逻辑异常抛出；
3. JWT工具类：实现token生成/解析/验证，集成角色鉴权（学生/辅导员/管理员）；
4. 阿里云OSS工具类：实现图片/文件上传（适配打卡、请假证明材料）；
5. 邮件发送工具类：实现考试/打卡提醒、请假审批结果推送；
6. WebSocket核心组件：
   - 配置类：WebSocketConfig（注册端点、拦截器）；
   - 服务类：WebSocketService（实现消息推送、连接管理、用户绑定）；
   - 处理器：WebSocketHandler（处理消息收发、连接建立/关闭）；
   - 核心逻辑：按用户ID推送消息，支持多场景实时提醒。

##### （二）课程&考试模块（管理员增删改查，学生查看本人数据）
1. 表结构SQL：生成课程表/考试表完整创建语句（含班级编号字段）；
2. 核心功能：
   - 管理员：课程/考试增删改查（分页+条件）、状态启用/禁用、按班级关联课程/考试；
   - 学生：仅查看本人班级的课程/考试列表（分页）、接收实时提醒；
   - 系统提醒：学生登录时，WebSocket推送「一周内的考试+24小时内的课程」，提示格式：「【考试提醒】您有《高等数学》考试将于3天后9:00在1教101进行」；
3. 接口：分页查询、单条查询、新增、修改、删除、提醒设置接口（JWT鉴权）。

##### （三）学业打卡模块（学生打卡，辅导员查看本班数据）
1. 表结构SQL：生成打卡表完整创建语句（含班级编号）；
2. 核心功能：
   - 学生：打卡（定位+图片上传至OSS）、查看个人打卡历史；
   - 辅导员：查看所负责班级学生打卡列表（分页+条件）、统计打卡率；
3. 接口：打卡提交、打卡记录查询（个人/辅导员）、打卡统计接口。

##### （四）请假模块（学生申请，辅导员审批，实时提醒）
1. 表结构SQL：生成请假表完整创建语句（含班级编号）；
2. 核心功能：
   - 学生：提交请假申请（上传证明材料）、查看审批状态/历史记录；
   - 辅导员：查看本班待审批请假列表、审批/驳回（填写意见）、查看本班请假记录；
   - WebSocket提醒：
     - 学生提交请假：推送提醒给对应辅导员（「有1条新的请假申请待审批」）；
     - 辅导员审批/驳回：推送提醒给对应学生（「您的请假申请已通过/驳回，意见：XXX」）；
3. 接口：请假申请提交、审批状态查询、辅导员审批操作、请假记录查询接口。

##### （五）AI智能咨询模块（学生专属对话）
1. 表结构SQL：生成咨询记录表（含学生ID、咨询问题、AI回复、时间）；
2. 核心功能：
   - 学生：提交咨询问题，调用真实AI接口（如讯飞星火/百度文心）返回专属解答；
   - 接口：咨询提交、历史记录查询接口（仅查看本人咨询记录）。

##### （六）消息&私信模块（WebSocket实时交互）
1. 表结构SQL：生成优化后的消息表完整创建语句；
2. 核心功能：
   - 学生私信：两个学生间互发私信，WebSocket实时推送，支持已读/未读标记；
   - 互动提醒：用户点赞/评论帖子时，WebSocket推送提醒给帖子作者（「XXX点赞了您的帖子《二手书转让》」）；
   - 系统提醒：学生登录时推送课程/考试提醒，显示未读消息数；
   - 核心逻辑：私信标记已读/未读，未读消息在前端显示红点提示；
3. 接口：消息列表查询（按类型/已读状态）、单条消息查询、标记已读、私信发送接口。

##### （七）管理员数据统计（选做）
1. 核心功能：统计打卡总数/率、请假总数/审批率、发帖总数（按时间/班级/分类）；
2. 接口：数据统计接口（返回JSON，适配前端可视化）。

#### 输出要求（后端）
1. 按模块输出：表结构SQL → Entity+DTO/VO → Mapper → Service+Impl → Controller；
2. WebSocket组件需完整，实现所有场景的实时提醒；
3. 接口需JWT鉴权，数据权限严格隔离（学生仅看本人数据）；
4. 代码可直接整合到现有项目，无需修改包名/配置；
5. 关键接口示例：
   - 学生打卡：POST /api/student/check/add（上传图片+定位，JWT鉴权）；
   - 辅导员审批请假：POST /api/counselor/leave/approve/{id}（传审批状态+意见）；
   - WebSocket推送：按用户ID推送消息，端点：/ws/message。

### 二、前端代码生成需求（匹配后端，严格适配样式）
#### 1. 基础环境（严格适配）
- Node.js版本：v20.14.0（原生支持ESModule）；
- Vue版本：Vue 3.4.x（组合式API + `<script setup>`语法糖）；
- 构建工具：Vite 5.x；
- 语言：JavaScript（非TypeScript）；
- 状态管理：Pinia 2.x + 持久化插件（存储登录信息）；
- 接口请求：使用项目已封装的request.js（import request from '@/utils/request.js'，前缀/api）。

#### 2. 项目技术栈&样式规范
- 角色样式区分：
  - 学生端：不使用Element Plus，自定义大气/高级/简约的PC端样式（白+浅灰为主，紫色点缀）；
  - 辅导员/管理员端：使用Element Plus，专业规整的后台风格；
- 样式：Scoped CSS + Less，响应式适配PC端；
- 路由：Vue Router 4.x，按角色拦截（学生不可访问辅导员/管理员页面）；
- 目录规范：
  - 学生端：src/views/student/；
  - 辅导员端：src/views/counselor/；
  - 管理员端：src/views/admin/；
  - 通用组件：src/components/；
  - Pinia：src/store/。

#### 3. 页面原型样式（必须严格遵循）
##### （1）学生端首页（无Element Plus）
- 布局：顶部导航栏 + 轮播公告区 + 功能入口区 + 提示区 + AI小助手悬浮按钮；
- 顶部导航：左侧「高校学生智能系统」（紫色），右侧消息图标（未读红点）、学号/姓名、头像下拉；

- 功能入口：4个圆润按钮（打卡/请假/论坛/课程考试），hover放大；
- 提示区：2个矩形卡片（课程提示/考试提示），显示最近课程/考试；
- AI小助手：右下角圆形按钮（机器人图标），点击弹出咨询对话框；
- 风格：简约大气，无冗余元素，紫色点缀。

##### （2）学生端帖子列表页（无Element Plus）

##### （3）辅导员端页面（Element Plus）
- 布局：左侧深色侧边栏 + 右侧内容区 + 顶部导航；
- 侧边栏：系统名称、菜单（请假审批/打卡统计），选中项高亮；
- 内容区：Element Plus表格（待审批请假/打卡数据），带分页/筛选；
- 风格：专业规整，操作便捷。

##### （4）管理员端页面（Element Plus）
- 布局：左侧深色侧边栏 + 右侧内容区 + 顶部导航；
- 侧边栏：系统名称、菜单（帖子管理/公告管理/用户管理/课程考试管理）；
- 内容区：Element Plus表格（带修改/删除/禁用按钮），分页/条件搜索；
- 风格：专业大气，逻辑清晰。

#### 4. 功能需求（100%匹配后端接口）
- 学生端：首页、帖子列表/发布、打卡、请假申请、课程考试查看（本人数据）、AI咨询、私信（实时收发+已读/未读）、WebSocket提醒弹窗；

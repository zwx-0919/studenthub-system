完整优化需求（前端界面美化 + 前后端统一 + 功能完善）
1. AI 助手界面
删除 AI 小助手每条回复消息的时间显示。
删除界面上的重新提问按钮，保留正常对话发送与展示。
对话气泡间距、内边距统一调整，界面更整洁，不改变原有布局结构。
2. 学生端界面补充
考试界面增加显示考试时间。
课程列表与课程详情增加上课时间、上课地点。
所有列表文字对齐、间距统一，避免拥挤或留白过大。
3. 后端 Redis 缓存全覆盖优化
用户管理、用户查询、用户信息全部走 Redis 缓存。
帖子列表、评论列表、公告、课程、考试、打卡记录等高频查询全部加缓存。
数据新增 / 修改 / 删除时自动清理对应缓存，保证数据一致。
防止缓存穿透、空数据缓存。
4. 前后端字段与功能统一
全面检索前端代码，删除后端不存在的接口、字段、功能按钮。
确保前端所有展示字段、请求接口后端都有，无报错、无无效渲染。
修复控制台 404、字段 undefined 问题。
5. 大数据量列表统一分页
所有数据量大的列表统一使用后端分页。
每页固定显示 10 条数据。
分页条样式统一、位置合理，不遮挡内容，不改变原有页面结构。
6. 帖子点赞功能完善
一个用户对一个帖子只能点赞一次，再次点击取消点赞。
点赞后红心标红高亮，刷新页面仍保持红色。
点赞数实时同步，样式统一美观。
7. 管理员端调整
移除管理员端新增帖子功能，隐藏按钮与入口。
公告支持上传多张图片。
学生端公告展示规则：
仅 1 张图片 → 正常显示；
≥2 张图片 → 不显示任何图片。
8. 辅导员负责班级显示格式
显示规则：专业名称：班级号1、班级号2、班级号3
示例：软件工程：220408班、220409班、220410班
排版整齐，不换行错乱。
9. 用户头像展示规则
学生、辅导员：优先显示真实上传头像；
未上传头像 → 显示姓名第一个字头像；
管理员：始终显示姓名首字头像，不变；
所有页面头像大小、圆形、间距统一规范，更美观。
10. 评论增加回复功能
支持对评论进行二级回复，展示格式：回复 XXX：内容
回复缩进、层级清晰，不混乱。
发送、展示、删除逻辑正常，有权限控制。
11. 学业打卡功能增强
打卡增加学习时长记录与展示。
新增每周 / 每月打卡与学习时长预警机制：
学生不达标 → 前端弹出提醒；
教师 / 辅导员后台收到预警名单。
学习记录列表排版整齐，字段对齐。
12. 管理员 & 教师端新增数据统计与可视化
增加统计：打卡次数、学习时长、班级完成率、预警人数排行。
使用折线图、柱状图、饼图展示，图表大小适中。
页面布局合理，不拥挤、不突兀，不破坏原有风格。
13. 前端界面整体优化（重点新增）
在不改变原有整体布局、主体结构、风格的前提下，对所有页面做以下优化：
统一页面内边距、外边距，去除拥挤、重叠、错位问题。
按钮大小、颜色、圆角统一， hover 效果一致。
表格、列表行高统一，文字垂直居中，隔行浅色背景更易读。
表单输入框、选择框对齐整齐，label 文字统一间距。
弹窗层级、遮罩、关闭按钮位置规范，不挡内容。
导航栏、侧边栏选中状态高亮更明显，提升体验。
卡片阴影、圆角适度增强，整体更精致、顺眼。
修复移动端 / 不同分辨率下的错位问题。
所有空数据页面统一展示 “暂无数据”，样式统一。
加载状态、成功 / 失败提示样式统一，不突兀。
14. 全局要求
所有修改不破坏原有功能逻辑。
不大幅改动原有样式结构，只做美化、对齐、统一、整洁。
前后端严格一致，无多余字段、无无效接口。
权限控制不变，仅优化展示与体验。

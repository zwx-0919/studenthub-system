## 1\. 项目概述

### 1\.1 项目基本信息

- 项目类型：全栈开发 / 毕业设计

- 开发角色：全栈开发、架构设计

- 核心目标：构建面向高校三类角色的一站式智能校园服务平台，实现业务线上化、服务智能化

### 1\.2 项目简介

本项目是一款面向高校学生、辅导员、管理员三类角色的全栈智能服务平台，融合传统校园业务模块与 AI 智能咨询能力，覆盖学习、事务、服务、社交四大核心场景，打通校园服务全业务闭环，提升校园管理效率与师生使用体验，为全校师生提供便捷、高效、智能的校园生活解决方案。

## 2\. 技术栈详情

采用主流全栈技术架构，兼顾性能、可扩展性与开发效率，具体技术栈如下：

### 2\.1 后端技术

- 核心框架：Spring Boot 2\.7\+

- 数据访问：MyBatis\-Plus

- 身份认证：JWT（JSON Web Token）

- AI 能力：Spring AI Alibaba、通义千问大模型

- 缓存技术：Redis

- 实时通信：WebSocket

- 构建工具：Maven

### 2\.2 数据存储

- 关系型数据库：MySQL 8\.0\+

- 缓存数据库：Redis 6\.0\+

### 2\.3 开发工具与效率

- 开发工具：IntelliJ IDEA、Cursor

- 高效开发：Vibe Coding

## 3\. 核心功能模块

系统基于角色权限设计，分为学生端、辅导员端、管理员端，各角色功能独立且联动，实现全业务闭环。

### 3\.1 学生端

- 用户认证：登录、注册、邮箱绑定校验

- 个人中心：个人信息查询与修改

- 学习管理：课程信息查询、考试安排查询

- 事务办理：学业打卡、请假申请、请假状态查询

- 智能服务：AI 智能咨询（多轮对话、实时信息查询）

- 社交互动：校园帖子发布、评论、互动

- 消息通知：实时接收审批结果、公告等通知

### 3\.2 辅导员端

- 事务审批：学生请假申请审批

- 学生管理：学生学业打卡查看、学生信息查询

- 公告管理：发布校园相关公告

### 3\.3 管理员端

- 用户管理：学生、辅导员账号管理（新增、编辑、禁用）

- 教学管理：课程信息管理、考试安排管理

- 内容管理：校园帖子审核与管理、公告管理

- 系统管理：系统配置、数据统计与监控

### 3\.4 AI 智能咨询模块

- 自然语言问答：支持学业疑问、校园政策、生活服务等场景问答

- Function Calling：打通 AI 与后端业务接口，实现课程、考试、请假状态等实时信息查询

- ChatMemory 机制：基于 Redis 存储对话上下文，提升多轮交互连贯性

## 4\. 项目亮点与技术难点

### 4\.1 核心亮点

- 全栈架构设计：独立完成从架构设计到核心模块开发，实现全业务流程闭环，保证系统稳定性与可扩展性。

- AI 智能化集成：基于 Spring AI Alibaba 接入通义千问大模型，结合 Function Calling 打通业务接口，实现智能服务升级。

- 交互体验优化：设计 ChatMemory 对话记忆机制，利用 Redis 缓存上下文，提升 AI 多轮交互的流畅度与准确性。

- 开发效率提升：采用 Cursor \+ Vibe Coding 高效开发模式，优化开发流程，使项目迭代效率提升 40%。

- 高可用设计：通过 JWT 实现安全身份认证，Redis 优化缓存性能，WebSocket 实现实时消息推送，保障系统高并发场景下的稳定运行。

### 4\.2 技术难点与解决方案

- 难点1：AI 与业务系统的联动 → 解决方案：通过 Function Calling 机制，实现 AI 意图解析与后端接口的无缝对接，获取实时业务数据。

- 难点2：多轮对话上下文管理 → 解决方案：基于 Redis 设计 ChatMemory 存储策略，高效管理对话历史，提升交互连贯性。

- 难点3：多角色权限控制 → 解决方案：基于 JWT 与自定义拦截器，实现细粒度角色权限控制，保障系统数据安全。

## 5. 快速启动指南

### 5\.1 环境准备

- JDK：1\.8 及以上

- MySQL：8\.0 及以上

- Redis：6\.0 及以上

- Maven：3\.6 及以上

- IDE：IntelliJ IDEA / Cursor

### 5\.2 启动步骤

1. 克隆项目到本地：`git clone https://github\.com/你的用户名/项目仓库名\.git`

2. 创建 MySQL 数据库，命名为 `campus\_service`，执行 `resources/sql/` 下的数据库脚本，初始化数据表与测试数据。

3. 修改配置文件：


- 修改 `application\-dev\.yml` 中的 MySQL 连接信息（url、username、password）。

- 修改 Redis 连接信息（host、port、password，若有）。

- 配置 Spring AI Alibaba 与通义千问大模型密钥。

4. 使用 IDEA / Cursor 打开项目，刷新 Maven 依赖，确保依赖下载完成。

5. 运行项目主类 `CampusServiceApplication\.java`。

6. 项目启动成功后，访问：`http://localhost:8080`（端口可在配置文件中修改）。
